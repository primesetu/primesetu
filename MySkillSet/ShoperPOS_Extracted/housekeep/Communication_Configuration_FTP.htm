Communication Configuration for FTP

The Communication Configuration window is as shown.

Communication Type: FTP is selected, based on the selection on the previous window.

Configuration Name: Enter the name for the FTP configuration.

The grid displays the properties to be set to use FTP for communication. The properties are listed under Names and the specified values under Value.

Server: Enter the FTP server name.

Port (Def. 21): Enter the number of the port to be used for accessing the FTP server. Default port number used is 21.

User Name: Enter the user name to be used for accessing the FTP server.

Password: Enter the password to be used for accessing the FTP server.

Destn. Folder: Enter the name of the folder where files (reports) are to be copied. By default, the root directory of on FTP is selected. You need to have sufficient access permissions on this folder to create files.

Use Passive: Select Yes, if the specified FTP is in passive mode. Otherwise select No.

Use Binary: Select Yes, to set the mode of file transfer to binary. Otherwise select No.

Active: Select to make the communication type active.

Test Connection: Click to check the connectivity.

Ok: To save the configuration

Cancel: To cancel the values entered or to close the Communication Configuration window, as the case may be

 

Note: Here you cannot add or view any other communication configuration.
