Goods Receipts based on Stock No.  Grid Detail

The combination of the fields available in this grid changes as per the value provided. There are some common fields which are displayed irrespective of the values provided. This grid is populated with values if you load the data from a P.O./ PDT File/ PT File.

Each item with its required details for issue can be entered in the Item Details grid from the Direct Entry grid. The columns displayed in the grids can be configured here.

The Direct Entry grid is displayed.

The grid displays the item details, if selected. Otherwise, you may scan the stock number, enter the stock number, or press F2 in the Stock No. column to select items using the search window.

Stock No: Enter or Scan or Browse (F2) the item for stock number.

Item Description: Displays the description for the stock number entered.

Note: In case Enable Delivery from Remote Location is set to 1-Yes in System Parameter,under category Delivery Process, then an additional column Location will be available in the grid. When location as part of batch is enabled, Location Code field will be displayed.

PO Prefix, PO No, and Pending PO Qty: These fields appear only when the transaction type is Purchase and a PO is selected under 1.Select Purchase Order.

Doc Qty.: The quantity as displayed in the document (example, invoice), is entered here. If items are loaded from a PT File, then this quantity which is pre-filled cannot be altered.

Act Qty.: Enter the item wise quantity received. This field can be filled with LSQ set for the item depending on the parameter set in the Inwards category. This field is populated when physical verification is for an inward consignment which is identified by the unique combination of Supplier Id, Ref. Doc. No. and Ref. Date.

Note:

If the stock items are configured to include serial numbers during the inwards process, a pop-up menu is displayed after entering the actual quantity.

In System Parameter, the Item Tag can be configured up to five levels under Item Tag category.

Click here to view the Item Tag Information window.

The Item Tag Information window is displayed as shown.

Item Information:

Stock Number: The stock number selected is displayed.

Descr: The description of the item is displayed.

Item Qty.: The quantity taken into stock is displayed.

The serial no details are captured as per configuration.

OK: To accept the entries

Cancel: To revert to quantity to zero

Selling Price and Current Cost: Displays the values available in the items master for these two fields. If the transaction type is Purchase then, Purchase Price is displayed instead of Current Cost.  A drop-down list is displayed (based on Item Classification parameter settings) when multiple prices are available. Select the price from the list.

Value: Calculates and displays the product of the Act Qty and Purchase Cost.

Discount Rate and Discount Amount: Enter the items level discounts, if any, as an Amount or Rate to be applied on Value. If rate is entered then the amount is automatically calculated and displayed and vice versa.

Tax Rate and Tax Amount: Enter Item level tax either as Rate or Amount. The purchase tax is calculated on predefined rates as per the settings in the System Parameters.

Add-on Before Tax: Enter any additional value to be added to the cost before calculating the Tax. Tax is also calculated for this value.

Add-on After Tax: Enter any additional value to be added to the cost. Tax is not calculated on this value.

Deduction Before Tax: Enter any value to be subtracted from the cost. Tax is calculated on this value also and reduced from the Tax Amount.

Deduction After Tax: Enter any value to be subtracted from the cost. Tax is not calculated on this value.

Note: Selection of multiple Addons and Deductions can be configured in System Parameters.

Doc. Total: Displays the total amount arrived by adding or subtracting, as the case may be, the values of Discount, Tax, Add-On and Deductions.

DC/ Inv. Total: Enter the DC/ Invoice total amount in this field.

Note: If the amount is greater than Doc. Total amount then the following screen is displayed.

If the amount is lesser than Doc. Total amount then the following screen is displayed.
