Field(s) Selection: Click this option to select the columns to be displayed in addition to the existing columns from the dropdown list.

1.Add Tax to Cost: Check this box if taxes entered in the document are to be considered for computing the cost price.

Source Tax Code: Enter the source tax code number.

Tax, Discount, Add-On and Deduction values entered at line level is applicable to all sizes available in a particular line.

Product, Brand, Style, Color: Enter each of these or press F2 to open the browse window and select the required values.

Cost Price: Displays purchase price of the item classification selected.

Qty Total: The sum of the quantity entered in different sizes in a line is displayed here.

Value: Displays the calculated value based on the quantity and cost price of the item.

Tax Rate: Enter the rate of tax of the item.

Tax Amount: Displays the calculated value based on the value and rate of tax defined.

Total Value: Displays the calculated value based on the total value and tax amount.

Size Details: Only those sizes relevant to the combination of Item Class1 (Product), Item Class2 (Brand), Item SubClass1 (Style), Item SubClass2 (Colour) are highlighted and the user is allowed to enter quantity received against the respective sizes.

If there are more than one stock number in the items master for the given combination of Item Class1 (Product), Item Class2 (Brand), Item SubClass1 (Style), Item SubClass2 (Colour), then you get a pop-up window, where you can click on the check box to the left of the required stock number.

Total Quantity: Displays the total quantity of all the items in the grid.

Total Value: Displays the total value of the items in the grid. This is a non editable field.

Item Level Disc.: Displays the total value of item level discounts.

The remaining footer level values can be either system appropriated value or user entered value. In these fields if rate is entered then the amount is calculated automatically and displayed; and vice versa.

Doc Level Disc.: Enter the document level discount, if any.

Tax: Enter the tax, if any.

Deduction: Enter the deduction, if any.

Addon: Enter the deduction, if any.

Deduction and Addon can be before or after tax.

Net Value: Displays the net value of all the items after considering the discounts/ tax/ deductions/ add-ons at the document level, if any.

Size-wise Goods Inwards Entry

Header Information:

Transaction Type: Select the transaction type from the dropdown list. The list displays the values as Purchase, Transfer In, Approval Receipt and Miscellaneous Receipts, based on the system parameter setting.

Supplier Id:  This caption dynamically changes based on the transaction type selected.

  • For Purchase, the caption is Supplier ID, Supplier Name; for Transfer In, the caption is Chain Store, Chain Store Name and for the other two types, the caption is Customer ID, Customer Name.

  • Enter the value or press F2 to browse and select the required value. On pressing F2, a window pops up with the relevant list of codes and names. Choose the supplier id/ customer id.

Note: When Approval Returns is chosen as the transaction type, the buttons Load PDT File and Load PT File are not displayed

Supplier Name: Displays the supplier’s name based on the Supplier Id.

Reference Doc: Enter the reference number of the document received which could be an Invoice, DC etc., or press F2 or click (the PT File browse) to browse in this field and select the reference number of the document received.  Choose the Invoice or DC displayed against a specific supplier.

Document Date: Enter the date of reference document

Reason: Select the reason for inward. This field gets displayed for transaction types other than Purchase. Press F2 to display the catalogued reason codes so that the appropriate one for the current transaction type can be selected.

Document Remarks: Enter suitable remark.

Issue Prefix & Issue No: These fields are displayed when the transaction type is Approval Receipt. Approval receipt is made against an approval issue made earlier. So, enter the prefix and No. of the approval issue.

Document Prefix: Depending on the type of transaction the prefix is displayed automatically

Document Number: Displays incremental number for the selected Document

Load from PT File: Click this option to load item details from Text file to the Size-wise Entry Grid.

Load from PDT File: Click this option to load Item Details from Text file.
