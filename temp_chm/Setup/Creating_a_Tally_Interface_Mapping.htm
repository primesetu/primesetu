Creating a Tally Interface Mapping

On clicking Add button in the Tally Synchronisation Manager window, you get a new field to accept the map name.

Map Name: Enter the map name.

Ok: To proceed with the configuration process.

Cancel: To abort the configuration process.

The Tally Interface - Mapping window is displayed.

Map Name: The name entered for the mapping is displayed.

Note: Tally 9 needs to be open on the server, while you are adding a Mapping.

Tally:

Server IP: Enter the IP address or name of the server where Tally 9 is installed.

Port No.: Enter the port number where Tally 9 is configured.

Company: Select the Tally Company Name to which Shoper 9 data has to be posted.

Note: In case you are using a firewall or proxy, please contact your network administrator for necessary configuration changes.

Sales Ledgers with Shop Name: Select if you want to post all Shoper 9 sales ledgers with shop name to Tally 9.

Post Paymodes Individually: Select if you want to post all Shoper 9 pay modes individually to Tally 9.

Post Deductions Individually: Select if you want to post all deduction recorded in Shoper 9 individually to Tally 9.

Post Add-Ons Individually: Select if you want to post all add-ons recorded in Shoper 9 individually to Tally 9.

Note: All the deductions and add-ons before tax will be factored in to the sale value or purchase value. All the deductions and add-ons after tax are displayed individually, if selected.

Post Inventory: Select if your statutory reporting requires HSN based report. This enables posting the complete inventory information from Shoper 9 to Tally 9. This option is available only if the Tally company chosen is VAT enabled. Note that HSN codes will be used only for inventory posting. For more information on HSN, click here.

Post Transfers: Select if your statutory reporting requires details of transfer-in and transfer-out.

Use Billing Remarks for Narration: Select this option to post billing remarks as narration in Tally vouchers.

Post Before Tax Add-Ons and Deduction Details: Select if you want to post pre tax calculation values of Add-ons and Deductions

Use Map Name in Voucher Number: Select to prefix Map Name to the voucher numbers posted to Tally 9.

Use Map Name in Bill References: Select to prefix Map Name to Bill References posted to Tally 9.

Do not Create Customer Ledgers: Select to disallow creating Customer ledgers in Tally 9.

Create non Transacted Party Ledgers: Select to post party ledgers that are created but not used in any transaction in Shoper 9.

Use Store Code as Cost Center: Select to use showroom code of shoper 9 company as cost centres in Tally 9.

The option Use Store Code as Cost Centre was controlled through system parameters in older versions of Shoper. This has been included to the mapping screen to provide single access point to options related Tally Interface Mapping.

Send Closing Stock: Select this option to post closing stock from Shoper.

Advanced Settings:

Offline Posting: Select if you want to post transaction into Tally 9 offline, i.e., when Tally 9 is not on the same network.

Deactivate: Select if you want to deactivate the selected mapping, when you are editing an existing mapping. Once deactivated, the mapping cannot be used for posting. you have to reactivate when posting is required. On reactivation the posting will happen from the last posted date.

Automatic Posting: Select to enable data posting as part of scheduled activity/ day-end.

Transactions to be sent:

Sales: Select No Posting, Cumulative or Bill by Bill to specify how sales transactions in Shoper 9 have to be posted to Tally 9. Whatever be your choice, credit sales are posted bill by bill.

Sales Returns: Select No Posting or Bill by Bill to specify how sales returns recorded in Shoper 9 have to be posted to Tally 9.

Purchase: Select No Posting or Bill by Bill to specify how purchase transactions in Shoper 9 have to be posted to Tally 9.

Purchase Returns: Select No Posting or Bill by Bill to specify how purchase returns recorded in Shoper 9 have to be posted to Tally 9.

Receipts: Select No Posting, Cumulative or Bill by Bill to specify how receipts in Shoper 9 have to be posted to Tally 9.

Payments: Select No Posting, Cumulative or Bill by Bill to specify how payments in Shoper 9 have to be posted to Tally 9.

Note: Cumulative postings will remove bill references when posting to Tally 9 - setting these to On Account. You might consider using Bill-by-Bill instead to ensure that bill references are posted to Tally 9.

Post Tax: Select the tax type to be applied as No Tax, Simple Tax, VAT Composition or VAT Regular.

Caution: It is advised that you do not change these tax settings. If the configuration done in Tally 9 is incorrect, cancel the mapping process, make the necessary changes in Tally 9 and redo the mapping process. Statutory reporting will be compromised if this warning is ignored.

 

Note: A trader coming under composite tax will not charge tax from his customers, and will set VAT to 0.00%. To ensure that Tally 9 processes composite VAT returns correctly, the appropriate tax classification has to be selected from the list for the sales ledger at the time of defining the mapping rules for the particular company.

 

Caution: Once mapped the tax classification cannot be changed in this mapping.

 

Post From: Select the date from when Shoper 9 data needs to be posted to Tally 9, as per the specified mapping. This date cannot be one later than the current date of Shoper 9.

Account Names in Tally:

Ledger Format: Select the ledger format to be used as MasterName_MasterCode, MasterCode_MasterName or MasterName while creating account names in Tally 9.

Select Prefix or Suffix to specify the string to be attached to account names posted to Tally 9 from Shoper 9. Select the string to be attached as Shoper, Shoper_CompanyCode (the company code prefixed with the string Shoper), CompanyCode (the company code) or <<Custom>>. If you choose <<Custom>>, a window is displayed to accept the text to be used in master name. The entered string is displayed next to the drop down.

Apply to Masters: Select if you want to apply the prefix or suffix to all masters posted from Shoper 9.

Apply to Voucher Types: Select if you want to apply the prefix or suffix to all voucher types posted from Shoper 9.

Auto Create Masters when Posting: Select if you want the masters unavailable in Tally 9 to be created automatically during posting. If you deselect this option, you would need to visit this setup again and post any new masters created in Shoper 9 to Tally 9.

Note: Each showroom can be mapped as a cost centre in Tally. Cost centres can be enabled or disabled in Shoper 9 POS using the parameter PostCCToTally under the category House Keeping. Once enabled, all postings are done using these cost centres.

 

Caution: Before posting data to Tally, cost centres need to be enabled in Tally. Subsequently, cost category and cost centres need to be created. Click here for more information.

 

Purchase Tax Breakup: To define the breakup of purchase tax when posting the values to Tally 9. Click here to learn more.

 

Note: Multiple tax components are enforced in certain states. Shoper 9 supports these multiple rates in Billing (Sales). However, multiple tax components are not supported in Purchase transactions and hence it is required that you use one single VAT% that combines the effect of all these components. You have to define the multiple tax components to ensure that the statutory reports in Tally 9 reflect the breakup correctly.

 

Preview Mapping: This option will display the list of masters that are going to be created in Tally 9. You will be able to change the ledger name, tax classification and first alias.

Save Settings: To save the configurations. An activities log will be displayed in case you are configuring a mapping for a group. On successful completion a message is displayed indicating that the mapping is successful.

Exit: To quit the mapping configuration option.

 

Note: Where a customer does not provide a 'C' form, an additional tax (equal to the regular VAT%) may be applicable for certain states in India. The additional tax needs to be a part of the VAT return (on a Rs 0.00 assessable value). This is currently not supported. The user is advised to make the necessary changes in such vouchers once they are posted into Tally 9.
