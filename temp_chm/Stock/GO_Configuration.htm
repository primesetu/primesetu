Configuration

Permitted Operations for Goods Outwards

To enable a functionality, set the parameter Permitted Operations for Goods Outwards under the category Outwards accordingly.

Transaction Types

To add values to the transaction types list, set the system parameter Valid Transactions in Goods Outwards under Outwards.

Reason Code

To add values under reason code, insert values under the category Reason for Misc Issues in General Lookup.

Load From PDT File

To enable loading from PDT file and to specify the format of PDT files, set the system parameter Allow PDT File Loading in Goods Inwards/Goods Outwards/Physical Verification under Inwards.

The following configurations are available under the Goods Outwards option:

Option

Value

Effect

PDT Loading Applicable

Y

Allows PDT file loading

N

Disallow PDT file loading

Default Format Applicable

Y

Allows to specify a constant file format under the option, Default File Format

N

Need to choose the file format at the time of loading the file

Default File Format (applicable only if Default Format Applicable is set to Y)

Stock No

2

Stock No and ActQty

3

Stock No, Rate and ActQty

4

Stock No, DocQty and ActQty

5

Stock No, Rate, DocQty and ActQty

Loading Order

1

Load Inward to Outward

To enable loading details from inwards document to outward documents, set the system parameter Import Goods Inwards into Goods Outwards under the category Outwards.

Enabling LSQ

To enable LSQ, set these parameters under the category Outwards:

  • Least Saleable Quantity on Item Selection - Misc. Issue

  • Least Saleable Quantity on Item Selection in Purchase Returns

  • Least Saleable Quantity on Item Selection in Transfer Out

Validation of Transaction Quantity

To validate stock during goods outwards, set these parameters under the category Outwards:

Stock check in Goods Outwards - Misc. Issue

Stock check in Goods Outwards - Purchase Returns

Stock check in Goods Outwards - Transfer Out

Configurations allowed are:

Value

Effect

0

Never check stock

1

Allow outward with warning

2

Disallow outward

3

Stop outward update, if multiple users are simultaneously doing outward and some of them do not find sufficient quantity for transaction
