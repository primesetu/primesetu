Discount on Item Combinations

This promotion type helps you to define schemes where buy a combination of different items and get some discount. Here, the definition is based on item combinations.

The details to be captured can be divided into two parts, namely, Buy Items and Discount Details.

Buy Items

This section allows you to select the item combinations to be sold to avail discount.

The Buy Items section is as shown.

Item Rate/Worth: Select if the definition is based on item rate or item worth. When you select this, the possible comparison operators that can be used to specify the selection criteria are filled in the list. The choices are Rate =, Rate >, Rate <=, Rate Range, Worth =, Worth >, Worth <= and Worth Range. Select the appropriate one and the Buy Items grid changes accordingly. Click here to view the details, if the definition is based on item rate/ worth.

Configure: Click to configure the options for item selection. Click here for more information.

The Buy Items grid displays the item classification columns as configured. On pressing F2 in any item classification column, the corresponding browse window is displayed with the list of catalogued values to select from.

Note: In the Buy Items grid:

    • You can press F2 anywhere in the grid, irrespective of the order in which the item classification columns are present, to select values of any attribute. Click here to learn more about value selection.

     • You have the flexibility to select either one or multiple attributes in a row, to specify the required items.

   • If Stock No. of an item is selected, the remaining item classification columns for that stock number will be filled automatically.

As promo is defined on item combinations, you need to specify the item combination by grouping the items into different sets. Select the first set of items in the grid.

Quantity: Specify the quantity for the first set of items to avail the discount.

Set No.: Auto assigned depending on the condition selected in the next column.

Condition: Select And or Or to specify the criteria. Last row will always have End. When you select And, the discount is applicable if the buyer picks the items in both the current row and the next row. Selecting Or indicates the discount is applicable if the buyer picks the item either in the current row or the next row.

If the condition selected is And the item in the current row and next row will be in two different sets and the sets will be numbered accordingly. If the selection is Or, the items will be in the same set and both rows will be assigned the same set number.

In case you are selecting Or, the quantity for both item combinations in the set will remain the same and hence the quantity column in the next row will be non editable.

Notice that set numbers are assigned to the rows based on your selection in Condition.

Discount Details

This section allows you to capture discount details for the item combinations selected in Buy Items grid.

The Discount Details section is as shown.

In terms of: Select Percentage or Amount to specify how the discount is going to be calculated. If you select Percentage, the Amount column in the Get Discount grid will be disabled and vice-versa.

Buy Item: The quantity defined in the combination grid is computed and displayed as the quantity criteria to be satisfied to get discount.

Get Discount: Enter the percentage or amount to be given as discount for the selected items.

2. More>>: Click to capture more details, if required. This button toggles between 2. More>> and 2. Less<<.

The Discount Details section changes as shown.

The additional information captured is:

Applied on: You may configure the default behaviour for selecting lowest/highest priced item on which the discount is to be applied.

Select Lowest priced item, if the discount is to be applied on the lowest priced item among the items bought.

Select Highest priced item, if the discount is to be applied on the highest priced item among the items bought.
