SMRITI-OS v3.0 — SKELETON CHUNK 06: DATABASE STRATEGY
================================================================
Approach: INSPIRED SCHEMA (not a copy of Shoper 9)
Reason  : Field names/business logic preserved from S9 where practical,
          but redesigned for PostgreSQL, multi-store, cloud, offline sync.

─── WHY NOT AN EXACT COPY ──────────────────────────────────────
Issue        | Shoper 9 Schema          | Smriti OS Requirement
Legal        | Tally IP                 | Inspired redesign — no legal risk
Database     | SQL Server (MSSQL)       | PostgreSQL — different data types
Architecture | Single store, LAN only   | Multi-store, cloud + offline sync
Identifiers  | INT IDENTITY (sequential)| UUID for distributed systems
Sync         | No sync columns          | sync_status, synced_at on every table
Audit Trail  | No audit columns         | created_by, updated_by, timestamps
Multi-tenant | No company isolation     | company_id on every table
Soft Delete  | Hard deletes             | is_deleted flag for data safety

─── SCHEMA CONVERSION PLAN (MSSQL Profiler Approach) ───────────
Step | Action                                          | Output
1    | Extract schema from Shoper 9 MSSQL              | Table list + column defs
2    | Capture foreign key relationships               | Entity relationship map
3    | Extract stored procedures + business logic      | Business logic reference
4    | Run MSSQL Profiler during billing/GRN/transfer  | Exact queries per transaction
5    | Map MSSQL data types to PostgreSQL              | Data type mapping table
6    | Generate PostgreSQL DDL (CREATE TABLE)          | PostgreSQL schema DDL
7    | Add mandatory columns to every table            | Final Smriti OS schema
8    | Generate SQLAlchemy ORM models from schema      | FastAPI-ready Python models

─── DATA TYPE MAPPING (MSSQL → PostgreSQL) ─────────────────────
MSSQL Type       | PostgreSQL Type    | Notes
INT IDENTITY     | SERIAL or UUID     | UUID preferred for distributed systems
NVARCHAR(n)      | VARCHAR(n)         | Direct equivalent
NVARCHAR(MAX)    | TEXT               | Unlimited text
DATETIME         | TIMESTAMPTZ        | With timezone for multi-region
SMALLDATETIME    | TIMESTAMP          | Without timezone
DECIMAL(p,s)     | NUMERIC(p,s)       | Direct equivalent — preferred for money
FLOAT            | DOUBLE PRECISION   | Direct equivalent
BIT              | BOOLEAN            | true/false
IMAGE            | TEXT (S3 URL)      | Store files in S3, URL ref in DB
UNIQUEIDENTIFIER | UUID               | Direct equivalent
MONEY            | NUMERIC(18,2)      | NEVER use FLOAT for money
TINYINT          | SMALLINT           | Small integer values
VARBINARY        | BYTEA              | Binary data

─── MANDATORY COLUMNS (every table) ────────────────────────────
Column      | Type                        | Purpose
company_id  | UUID NOT NULL               | Multi-tenant isolation
store_id    | UUID (nullable for HO)      | Store level isolation
sync_status | VARCHAR                     | Values: pending/synced/conflict
synced_at   | TIMESTAMPTZ                 | Last successful sync timestamp
is_deleted  | BOOLEAN DEFAULT FALSE       | Soft delete — never hard delete
created_by  | UUID                        | User who created this record
updated_by  | UUID                        | User who last updated this record
created_at  | TIMESTAMPTZ DEFAULT NOW()   | Auto-set on insert
updated_at  | TIMESTAMPTZ DEFAULT NOW()   | Auto-updated on every change
uuid        | UUID DEFAULT gen_random_uuid()| Distributed unique identifier
