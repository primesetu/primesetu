Data Rebuild

This option is used when the database seems to have got corrupted which becomes evident from one of the following conditions When -

  • The reports are not being created properly.

  • There is inconsistency in the values of any item, say stock value, when printed in different reports.

  • The stocks of some items are being shown as negative even when the parameter Stock Out Action in Billing is set not to allow billing of the items without stock.

How to reach: Setup > Supervisory Functions > Data Rebuild

The Rebuild window is displayed.

 

Shoper 9 writes a counter number for all the transactions of each Shoper-date which is incremented for subsequent Shoper-date.

Update all VA Counter: Tick to update the counter number of all the transactions

All Transactions: Select to consider full database for rebuilding

From Selected Month and Year: Select to consider transactions from a particular month and year for the rebuilding process selectively.

Select Month and Year: choose the month and year from the list. This is activated only if From Selected Month and Year is selected.

Update: Click to start the data rebuild

On completion, the following message is displayed.

 

OK: To continue

Quit: To exit from the option
