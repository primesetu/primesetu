Node Management

Node Registry maintains the details of all nodes that can be used to access Shoper database.

This option allows you to define the nodes from which Shoper database can be accessed or you can deny nodes from accessing Shoper database. The Nodes can be defined only by a supervisor.

Note: If node wise restrictions are enabled for any node, then the same has to be done for all the nodes which have access to Shoper 9 HO. If node wise restrictions are not required, it is suggested that the node registry is left empty without defining any node.

The process of Node Definition retrieves the information from the node it is invoked from. This option can be used in any given node only to include that node in the node registry or to modify the details of that node.

How to reach: Setup > Supervisory Functions > Node Management

The Login window is displayed to confirm the supervisor's login ID and password.

User ID: Enter supervisor’s login ID (Default is super).

Password: Enter supervisor’s password (Default is nothing).

 

Ok: To open the Company Maintenance window

Cancel: To quit the process

The Registry Login window  is displayed.

 Local Node: Select to add, modify or delete a node registry.

Remote Nodes: Select to dissociate any of the client nodes while you are working in a multi-user environment.

Cancel: To quit the process

Next: To proceed with the node registration process

The next step of the node management process depends on your selection of

  • Local Node

    The Node Creation Form is displayed.

    Node options: Add Node option is selected, if the current node is not registered. If the node is already registered, this option will be disabled and a message Already Node Registry Done will be displayed at the bottom of the Node Creation window.

    Node Id: Enter the node id for the current system. Node id can be used meaningfully to represent the physical position of the system, segment of business, etc., if used appropriately.

    Node Name: Enter the name of the system. Appropriate naming conventions would make management of systems easier.

    Centre Cash Point: Set Value as Yes to enable the terminal as the centre cash collection terminal. At this terminal cash reconciliation of other terminals take place.

    Cash Collection Point: Set Value as Yes to enable the terminal as the cash collection terminal. At this terminal cash collection and inter terminal transfers take place.

    Ok: To complete the node registration process

    Exit: To quit the operation.

    Note: If you choose the option Modify Node, the node id and node name will be displayed but only the node name is editable.

  • Remote Nodes

    The Registry Login window is displayed.

    Remote Nodes: Node details of all the clients are displayed here if you are a supervisor logged in at the server. Click to select the node details that need to be denied access to the Shoper database.

    Delete: To deny access to the Shoper database for the node selected
