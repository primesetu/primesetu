POS to POS Communication Details

POS to POS communication facility is useful when two POS locations using Shoper 9 POS want to communicate to each other, without the help of Shoper 9 HO to mediate between the POS locations.

Scenario 1

There may be cases where showrooms are part of a chain and can receive data from HO. However, data communication is scheduled at longer intervals. For example, data from showrooms is synchronised to HO only at day-close. Then details of inter-showroom transfers will take almost a day to reach the destination, even if the stock reaches the destination immediately. In such cases this facility can be utilised for immediate data transfer. When data comes from HO as part of next synchronisation, the same file is not loaded a second time, as it is already loaded. Therefore, no data mismatch occurs.

Scenario 2

Various retail businesses using Shoper 9 operate as a group of four or five outlets, without a designated central management unit. Hence, they do not find it necessary to use Shoper 9 HO for central management and consolidation. However, there may be inter-showroom transactions like transfers. POS to POS communication facility is useful in such scenarios, to avoid manual intervention to transfer and load data. This facility also helps in ensuring data security by avoiding data modification outside Shoper 9.

POS to POS communication feature is different from the default synchronisation in Shoper 9. If detailed tracking of the transaction information and consolidation of data from multiple showrooms is required, presence of Shoper 9 HO is necessary.

This facility is for sending only transaction files (PT/ TT). These transactions include invoice (bill), transfer-in and transfer-out. As it supports transfer of PT files, item masters can be created in inwards using PT files.

In this window you can execute the following functions.

Add: Click to add communication details for a different Document Type and Mode combination. If a Document Type and Mode combination is already configured, then the configuration details are displayed when you select that particular combination. Click here for more information.

Edit: Click to modify an existing communication details specified for a  Document Type and Mode combination . Select the required Document Type and Mode combination and the details are displayed. Make the necessary modifications and click Apply to save the changes made.  

Delete: Click to delete an existing communication details. Select the required Document Type and Mode combination and the details are displayed. Click Apply to confirm deletion.

Exit: To quit the Configure Synchronisation option
