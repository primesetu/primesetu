Slips Configuration

Shoper 9 POS allows you to create different Slips (Expected Transactions) as per the bill type (Product/ Service) selected. Some of the commonly used slips in a POS environment are Suspension, Sales Advice Slips, Sales Order and Service Order. The parameters under the category Slips allow you to configure different fields available for each slip. Each parameter manages a set of fields relevant for each type of expected transaction document. You may enable/ disable each field in these documents independently.

The following table lists some of the commonly used parameters which can be configured to:

  • Accept the required header and item details while creating the respective slips

  • Choose the printing mode

Parameter

Value

(In each field for each document type)

Effect

Header Details Accepted In Expected Transactions

1

The corresponding field in the respective document will be enabled

0

The corresponding field in the respective document will be disabled

Expected Transaction Header Details Mandatory

1

The corresponding field in the respective document will be mandatory

0

The corresponding field in the respective document will not mandatory

Item Details Accepted in Expected Transactions

1

The corresponding field in the respective document will be enabled

0

The corresponding field in the respective document will be disabled

Expected Transaction Item Details Mandatory

1

The corresponding field in the respective document will be mandatory

0

The corresponding field in the respective document will not mandatory

Print Expected Transactions in Condensed Mode

1

All Expected Transactions will be printed in Condensed mode

0

All Expected Transactions will be printed in Normal mode
