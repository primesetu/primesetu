Additional Details in Customer Catalogue

The Other Details window is as shown. This window has two tabs—Tax & Loyalty and Payment & Transaction.

Tax & Loyalty tab

VAT/ CST Details:

LST/ VAT/ TIN: Enter the registration number and select the date of registration.

CST Number: Enter the registration number and select the date of registration.

Tax Type:

Destination Tax: Select the destination tax type. This is available only if the system parameter Destination-wise Tax Grouping Applicable under the category Tax is enabled. The values listed here are those catalogued in general lookup under Destination-wise Sales Tax Group.

Tax Form Details:

Pre-Sale Tax Form Applicable: Select, if pre-sale tax form is applicable.

Pre-Sale Form Name: Enter the pre-sale form name, if applicable.

Post-Sale Tax Form Applicable: Select, if post-sale tax form is applicable.

Post-Sale Form Name: Enter the post-sale form name, if applicable.

Loyalty Program Details:

Program Code: Enter the loyalty program code, if applicable.

Program Name: Enter the name of the loyalty program, if loyalty program is applicable.

Dependant Details:

Code: Select the code of the customer who has to be linked as a dependant.

Name: Displays the name of the selected customer.

Address: Select Same as Customer Address, if the dependant also has the same address as the customer being catalogued.

Payment & Transaction tab

Payment Details:

Category: Select Cash or On-Acc from the list.

Terms: Enter the payment terms.

Credit: The values for Limit and Days as set in the Customer Price Group are displayed. You may change these, if required.

Credit Used: Enter the credit used by the customer, if required.

Price Factor: Enter the Retail/ Buying and Dealer/ Selling factors (for example, 10) to be applied as a rate or amount on different prices while creating flat files for any type of transaction. Specify the place value to which the prices are to be rounded off.

Note: The factors set here are used in the option Setup > General Setup > Flat File Value.

Bank/ Transport Details:

Bank Code: Select the bank with which the customer does transactions. The values in the list are as catalogued in general lookup under List of Banks.

Bank Location: Enter the location of the bank.

Transport Mode: Select By-Hand or By-Road, as the mode of transport.

Transporter Code: Select the transporter. These values are as catalogued in the General Lookup as part of installation. You may change these as per requirement.

Transit Days: Enter the number of days required for transmitting the items.

Transactions Allowed: Select the transactions allowed for this customer. The transactions possible are Cash Bill, Credit Invoice, Misc. Issue, Misc. Receipt and DC Generation. You may select any or all of these transactions.

Shoper Details: The information under this heading is applicable if the customer, whose details are being catalogued, is also a Shoper user. These details help in making the data exchange between the customer's system and your system easier.

Company Code: Enter the Shoper company code as in the customer's system.

Environment: Select Retail or Distributor depending on the environment in which the customer operates.

Flat File Format: Select the appropriate format from the list, in which the flat files have to be created.

Delimiter: Enter the delimiter to be used in the creation of flat files for data exchange.

Tax Inclusive: Select if the item rates used in the customer's Shoper are inclusive of taxes.

Ok: To accept the values and close the window

Cancel: To ignore the values entered and close the window
