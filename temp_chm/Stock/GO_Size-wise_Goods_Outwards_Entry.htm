The features applicable to all transaction types under size-wise entry are:

    • Size column is highlighted based on the Product/ Brand combination selected.

    • The combo list is populated when the Cost Price of the items differ with in Product, Brand, Style and Colour combination.

    • The size grid is enabled based on the Product, Brand, Style, Colour and DocEnt Rate selected from the combo list.

    • The outward quantities of all sizes are entered in a single line in separate columns for each size.

    • A popup window is displayed to select the appropriate stock number when there are more then one SKU with same combination.

Note: In case of Purchase Returns, DocEntRate is considered along with Class1-Class2/Style and colour combination to enable the size grid; in case of Transfer Out and Miscellaneous Issue, Current Cost Price is considered along with Class1-Class2/Style and colour combination and in case of Approval Issue, Selling Price is considered along with Class1-Class2/Style and colour combination.

Total Quantity: Displays the total document quantity of the selected items in the items grid.

Total Value: Displays the total value of the items in the grid. This is a non editable field.

Doc Level – Rate/ Discount: Enter the document level discounts, if any, as amount or rate. If rate is entered, then the discount amount is calculated automatically and displayed; and vice versa.

Item Level: Displays the total value of item level discount.

Tax: Displays the total tax amount if entered at the item level in the grid. If not, the values can be provided at the document level.

The remaining footer level values displays the amount entered at the item level in the grid. If not, the values can be provided at the document level. In these fields if rate is entered then the amount is calculated automatically and displayed; and vice versa.

Deduction: Enter the deduction, if any.

Addon: Enter the addon, if any.

Deduction and Addon can be before or after tax.

Doc. Total: Displays the net value of all the items after considering the discounts/ tax/ deductions/ add-ons at the document level, if any.

Net Value: Displays the net value of all the items after considering the discounts/ tax/ deductions/ add-ons at the document level, if any.

A message is displayed suggesting an adjustment in the Net Value.

 

Goods Outwards Size-wise

Header Information:

Transaction Type: Select the transaction type from the dropdown list. The list displays the values as Purchase Returns, Transfer Out, Approval Issues and/ or Miscellaneous Issues, based on the system parameter setting.

Doc No. Prefix: Depending on the type of transaction the prefix is displayed automatically

Document No.: Displays incremental number for the selected Document

Reason Code: Select the reason for outward from the drop down list. This combo box has reason codes flowing from general lookup.

Party Code: Enter the party code based on the transaction type selected. Press F2 to display the party codes and name, select the appropriate party code.

SalesMan Cd: Select the salesman code.

Purchase Prefix and Purchase Doc No.: These fields are displayed only when the transaction type Purchase Returns is selected. Enter the purchase prefix and document Number.

Doc Remarks: Enter the document remarks.

1.Load from PDT File: Click this option to load Item Details from Text file.

2.Load Inward to Outward: Click to send all the items received against a goods inwards to any of the chain store. Enter the inward document prefix and number.

3.Load Dispatch Advice: This option is visible only if a Dispatch Advice is available.

4.Size-wise Entry: Select this box to switch the mode of entry from stock number to size-wise or from size-wise to stock number.

Product, Brand, Style, Color: Enter each of these or press F2 to open the browse window and select the required values.

Cost Price: Displays cost price of the item classification selected.

DocEntRate: Enter the transaction value. You can also select any of the previous transaction values, if available.

Note: If the Parameter Allow user selection for rate of issue(Outwards) under the category Outwards is not selected, the transaction rate of the last document entered under the selected type of transaction is displayed by default, which cannot be edited.

Qty Total: The sum of the quantity entered in different sizes in a line is displayed here.

Value: Displays the calculated value based on the quantity and cost price of the item.
