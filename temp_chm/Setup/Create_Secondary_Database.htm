Create Secondary Database

This option allows you to create the secondary database for storing Shoper 9 POS data of the current company.

The Head Office generally sends complete master records to its chain stores irrespective of the actual items that are available at the store to sell. If these master records are imported directly into the Shoper 9 POS database, the retrieval of the master record will be slow and it will impact the performance of Shoper 9 in general.

Shoper 9 therefore supports a ‘secondary’ database that will hold the information of all item masters, and selectively imports masters required by the chain store into the primary database.

How to reach: Setup > Secondary Database

The Manage Secondary Database window is displayed.

In this window you can execute the following functions.

Create Secondary Database: To create a new secondary database and attach it to the current open company. Click here for more information.

Attach Secondary Database: To attach an existing secondary database to the current open company. Click here for more information.

Migrate Secondary Database: To migrate an existing secondary database in an older version of shoper and attach it to the current open company. Click here for more information.

Cancel: To exit the Manage Secondary Database window.

Next: To go to the next window.

Advanced: To confirm/ change the details like Database Name, Server Name, User ID, Password, Data File path and Log File path to be stored.

The Advance window with details of the current company is displayed.

Ok: To confirm the default details or to save the changes made.

Cancel: To cancel the changes made, if any, and return to the previous window.
