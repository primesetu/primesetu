Stock No: Enter or Scan or Browse (F2) the item for stock number.

Item Description: Displays the description for the stock number entered.

PO Prefix, PO No, and Pending PO Qty: These fields appear only when the transaction type is Purchase and a PO is selected under 1.Select Purchase Order.

Doc Qty.: The quantity as displayed in the document (example, invoice), is entered here. If items are loaded from a PT File, then this quantity which is pre-filled cannot be altered.

Act Qty.: Enter the item wise quantity received. This field can be filled with LSQ set for the item depending on the parameter set in the Inwards category. This field is populated when physical verification is for an inward consignment which is identified by the unique combination of Supplier Id, Ref. Doc. No. and Ref. Date.

Selling Price and Current Cost: Displays the values available in the items master for these two fields.

Value: Calculates and displays the product of the Act Qty and Purchase Cost.

Discount Rate and Discount Amount: Enter the items level discounts, if any, as an Amount or Rate to be applied on Value. If rate is entered then the amount is automatically calculated and displayed and vice versa.

VAT Rate and VAT Amount: Enter Item level tax either as Rate or Amount.

Add-on Before Tax: Enter any additional value to be added to the cost before calculating the Tax. Tax is also calculated for this value.

Add-on After Tax: Enter any additional value to be added to the cost. Tax is not calculated on this value.

Deduction Before Tax: Enter any value to be subtracted from the cost. Tax is calculated on this value also and reduced from the Tax Amount.

Deduction After Tax: Enter any value to be subtracted from the cost. Tax is not calculated on this value.

Doc. Total: Displays the total amount arrived by adding or subtracting, as the case may be, the values of Discount, Tax, Add-On and Deductions.

DC/ Inv. Total: Enter the DC/ Invoice total amount in this field.

Note: If the amount is greater than Doc. Total amount then the following screen is displayed.

If the amount is lesser than Doc. Total amount then the following screen is displayed.

 

Total Doc Quantity: Displays the sum of Doc Qty of all the items in the grid.

Total Act Quantity: Displays the sum of Act Qty of all the items in the grid.

Total Value: Displays the total value of the items in the grid. This is a non editable field.

Doc Level – Rate/ Discount: Enter the document level discounts, if any, as amount or rate to be applied on Total Value. If rate is entered then the discount amount is calculated automatically and displayed; and vice versa.

Item Level: Displays the total value of item level discount. This is a non editable field.

Tax: Displays the total tax amount if entered at the item level in the grid. If not, the values can be provided at the document level.

The remaining footer level values can be either system appropriated value or user entered value. In these fields if rate is entered then the amount is calculated automatically and displayed; and vice versa.

Deduction: Enter the deduction, if any.

Addon: Enter the addon, if any.

Deduction and Addon can be before or after tax.

Doc. Total: Displays the net value of all the items after considering the discounts/ tax/ deductions/ add-ons at the document level, if any.

DC/Inv. Total: This field has to be entered by the operator referring the hard copy of the DC/ Invoice total. If there is mismatch between Doc. Total and DC/ Inv. Total, then the require adjustments have to be done in the Addon or Deduction field.

On scanning an item, the details of the item scrolls in the status bar.

1. Select Purchase Order: This is useful to select items against the pending Purchase Order’s.

Note: If a purchase order raised on the selected supplier is pending and the option 1.Select Purchase Order is not selected, then a message Purchase Order Found for Selected Vendor, Do you wish to Select PO is displayed. Click Yes to select the pending Purchase Order. If you select No the cursor moves to the Stock No. column and you can enter the stock number or select the stock number by pressing F2 as per the requirement.

2. Load from PT File: Click this option to load item details from PT File.

The item details in the file will be loaded on to items grid only if Supplier ID/ Chain Store ID, Doc Ref Number and Doc Date entered in the respective fields of inwards entry screen matches with that content of the PT file. If not, a proper message will be displayed. A log file (Pt_Validator) is created in the log folder (Shoper\Share\Log) while trying to load a PT File.  Indicators are provided in the log file to identify the errors and corrections. If the PT File cannot be loaded (due to format or data type errors), check this log file to verify and correct the errors.

A Goods Inwards done using Load from PT File creates the pricing master for new items and updates the stock master.
