Adding Customer Details

The Customer Details window is as shown.

Customer Name & Address:

Customer Code: Enter the value, if the system parameter Customer Code User Assigned under the category Customer is enabled. Otherwise, the code is generated and displayed automatically.

Enter values in Customer Name, Address, Locality, Country, State and City.

Price Group: Select a value from the list. Values in the list are those catalogued in the Customer Price Group option.

Phone & Email: Enter Office & Home phone, Mobile number, Fax number and Email id.

Additional Address: Click to add more addresses. This button is available only if multiple addresses are enabled.

The Mailing Address window is as shown.

Note: To enable multiple addresses in the mailing address window, add parameters under the category Types of Address in General Lookup. There can be any number of contact addresses.

The Customer Code and Customer Name are displayed in the Code and Name fields respectively. These cannot be altered. The address, if entered in the Customer Details window are displayed here.

The captions: Address1, Address2, Address3, Address4, Address5 and Default Address can be changed by setting the system parameters under the category Customer, if required.

Enter the details as required. You may set this address or one of the addresses (if you have more than one parameter configured), as Default Address. This address would be used for all correspondences.

Ok: To save the mailing details of the customer and close the window

Cancel: To close the window without saving the details entered

Personal Details:

Birth Day: Select the format DD/MM/YYYY (if the date includes year) or DD-MMM (if only day and month) to identify the format in which you are entering the date. Select a date in the next field.

Gender: Select M or F.

Married:  Select, if the customer is married, then enter the anniversary date. If the customer is not married, you have an option to enter the wedding day. In both the cases, select the format DD/MM/YYYY (if the date includes year) or DD-MMM (if only day and month) to identify the format in which you are entering the date. Select a date in the next field.

More: Click to add classification and profile details of the customer, if required. The button caption changes to Less.

Shoper 9 has provision to define five customer classification attributes and five customer profile attributes. You may use some or all of these and specify captions for the required ones.

These attributes can be configured as part of installation using templates or by setting the system parameters under the category Setup. The values for these attributes are catalogued in General Lookup.

Select the required values.

 

Other Details: Click to enter details of tax, loyalty program, payment and transaction.  Click here for details.

Note:

• If you have any extension plugged-in for capturing data from other sources, a button named Get Custom Details appears to the right of Other Details. Click the button to use the plug-in.

• In case you have an extension that does validation of the data entered on the screen, the same gets triggered when you click Ok. This extension can return an indicator to the catalogue program allowing to save the data, or forcing to cancel saving. This program can also display error messages related to  validation.

 

Ok: To confirm and save the details of the customer. If customer code is auto-generated, the code assigned to the customer is displayed for future reference.

Cancel: To clear the details entered.
