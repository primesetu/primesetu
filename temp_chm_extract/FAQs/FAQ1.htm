FAQs on Extensions

  1. What are extensions?

Extensions are any customised add-ons to the base product (here, Shoper 9 POS). Extensions that can be added in Shoper 9 POS are used to meet the requirements specific to individual business scenarios. In general, extensions are used for customisations of the standard Shoper 9 POS functionalities.

  1. Where can I implement extensions?

Extensions can be implemented in all transactions and catalogues of Shoper 9 POS.

  1. How can I implement extensions?

Implementation of extensions can be summarised in three steps as:

    • Creating Custom DLL

    • Deploying Custom DLL

    • Configuring Custom DLL details in GenLookUp table

  1. What can be done with extensions?

    • Extensions can capture additional values and save the data in Shoper 9 database

    • Extensions can decide whether the customised data is mandatory or not

    • Extensions can be displayed before confirming the transactions

    • Multiple extensions can be enabled at any transaction level

    • Customised data can be edited

    • Extensions can be set active or inactive

    • Customised data can be displayed in the transaction program

    • Validation of the extensions can be handled in three ways

    • No validation

    • Validation at Custom DLL

    • Validation at the transaction program

  1. What are the limitations of extensions?

    • Extensions cannot update the standard Shoper 9 values

    • Extensions cannot change the look and feel of the standard Shoper 9 application UI

    • Data cannot be printed

  1. Can you give some examples where extensions can be used?

Some sample extensions that are supported in Shoper 9 POS are:

    • Handling credit card validation

    • Updating credit limit

    • Updating loyalty points

    • Handling gift coupons and Sodexo coupons

    • Capturing driver information in transporter details

    • Capturing mobile details (IEMI No, Battery No, Insurance No)

    • Capturing commission value for Sales person

    • Displaying information on related accessories while customer buying an item

  1. Where is the customised data saved?

Additional fields are provided in the relevant data tables to store the data captured for extensions.

The details of the additional fields available for extensions in SaleTrnHdr, StkTrnDtls, StkTrnHdr, PosCashTrn tables are:

    • Provision for Eight string values - to capture string values

    • Provision for Four integer values - to capture integer values

    • Provision for Four money values - to capture money values

    • Provision for Four date time values - to capture date & time details

These values are saved in Shoper database.

Additionally, you can create customised tables to store extension details, if required. In case you create customised tables, you need to make sure that the tables contain the following fields and their values while saving the records.

  1. What will happen to extensions when I upgrade Shoper?

Extensions will continue working unless there is any change in the Extension Framework. But if there are any changes in the Extension Framework, the Partner has to make the required changes in the extensions accordingly.

  1. Can extensions developed for one user is used for other users?

Extensions can be accessed from the Extension Framework only if you are authorised to access the same, by the Partner. In other words, you can access any extensions only if your Account ID is authorised by the partner to access the extensions.

  1. What is the difference between extension and customization?

Extensions are customised add-ons to an exiting functionality (e.g., Billing, Inwards, etc.) to extend its functionality. Extensions are controlled by Extension Framework.

Customisations are individual custom programs to solve customer specific requirements (e.g., New reports in addition to the standard Shoper 9 reports).
