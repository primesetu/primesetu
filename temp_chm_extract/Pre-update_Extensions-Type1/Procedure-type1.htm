Transaction Extensions – Procedure

The steps involved in the process to create Transaction Extensions can be summarised as:

  1. Check whether the extensions are available

  2. Shoper 9 application passes information to Framework depends on the transaction level at which the extensions are enabled

  3. Framework creates intermediate temp table and stores the information received from the application in it.

Note: Temp table will have standard fields and custom fields that are reserved for customisations or extensions.

  1. Framework sends the intermediate temp table to extensions (custom programs)

  2. Extension can validate standard fields in the temp table

  3. Extensions (custom program) take decisions based on validation, i.e., whether to continue or not

    1. Custom program can capture additional information and update it in the intermediate temp table

    2. Information in the temp table is passed to the framework

    3. Framework to the Shoper application

    4. Shoper application saves the information in the custom fields of the standard table

Procedure to Create Transaction Extensions

There are three steps to create an extension supported by Extension Framework in Shoper 9:

  • Creating Custom DLL

  • Deploying Custom DLL

  • Configuring Custom DLL details in GenLookUp table

Creating Custom DLL and deploying it at POS location

To know how to create and deploy Custom DLL, Refer Appendix

Mandatory function to be present in a Custom DLL is:

Public Function ShowDllForm1(StrTableName As String, StrSqlConnstr As String) As Integer

There are two arguments in the ShowDllForm1 function and are explained in the following table.

Remarks

In GenLookUp table, Code is inserted as a combination of Transaction Type, Extension level and Extension Invocation order each separated by a ‘+’ symbol. For example, if the Transaction Type is 1100, Extension level is 1 and Extension Invocation order is 1, then the corresponding Code is 1100+1+1.

Alternatively, you can use Extension Script Generator Tool to generate the corresponding script which contains the SQL statements to be inserted in GenLookup table for configuring the required custom DLLs.

Note: If there are multiple extensions at different levels of a transaction, extensions should be called in sequential order within each transaction level. For example, consider the following configuration details from GenLookUp table for a billing transaction.

The numbers of extensions at different levels of the given billing transaction are:

  • Header Level – 3 Extensions (i.e., 2100+1+1, 2100+1+2 and 2100+1+3)

  • Detail Level – 2 Extensions (i.e., 2100+2+1, 2100+2+2)

  • Footer Level – 1 Extension (i.e., 2100+3+1)

  • Footer Detail Level – 1 Extension (i.e., 2100+4+1)
