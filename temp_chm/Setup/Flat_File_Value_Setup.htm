Flat File Value Setup

This option is used to apply the settings while creating the flat files. These settings are useful if the Vendor or the Customer or the Chain Store also has Shoper with same item classification as that of the sender.

How to reach: Setup > General > Flat File Value

The Flat File Values Settings window is displayed.

 

The values set here will be used for creating flat files for the seven transactions types mentioned. Select Retail Environment or Distributor Environment depending on the environment of the receiver of the flat file.

The five prices written in the flat file are in the order of Retail_Price, Dealer_Price, CurrentCost, LastPurchPrice and DocEntRate, by default. The positions of these prices can be interchanged as required.

 

In the above figure you may select Retail Price from the drop down list so that Retail Price is written in the place of Current Cost while creating flat files with respect to 2100-Sales Invoice for all the customers.

Check the relevant box in the column Factor Applicable to apply the Selling Factor or Buying Factor on any of the five prices with respect to any transaction type.

 

In the pop-up window, select Add/ Subtract to apply the Buying or Selling factor (as entered while cataloguing the Vendor or the Customer or the Chain Store) as a Rate/ Amount for any or all the five prices displayed under Factor Value. Click OK to accept the selections.

Save: To save the settings

Exit: To quit the option
