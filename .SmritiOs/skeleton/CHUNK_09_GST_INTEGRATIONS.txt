─── MODULE 13: GST & TAX COMPLIANCE ────────────────────────────
Coverage:
  Multi-GSTIN support per store/branch
  Tax: CGST, SGST, IGST, UTGST, Cess (auto-determined by transaction)
  HSN/SAC mandatory on every item — searchable HSN master
  GST Slabs: 0%, 5%, 12%, 18%, 28% + Cess (configurable per item)
  Composition Scheme support
  Reverse Charge Mechanism (RCM)
  ITC tracking + GSTR-2A/2B reconciliation

TAX DETERMINATION LOGIC:
  Same State (Intra-state)        → CGST + SGST
  Different State (Inter-state)   → IGST
  Union Territory                 → CGST + UTGST
  Unregistered Buyer (B2C)        → B2C rules, no ITC
  Composition Dealer              → Fixed % on turnover
  Reverse Charge Mechanism        → Buyer pays tax

GST RETURNS SUPPORT:
  GSTR-1   | Monthly/Quarterly | All sales invoices & credit notes
  GSTR-3B  | Monthly           | Summary tax liability
  GSTR-2A/2B | Monthly         | Purchase reconciliation
  GSTR-9   | Annual            | Full year consolidated
  CMP-08   | Quarterly         | Composition dealers only

GST FILING METHODS:
  GSP (GST Suvidha Provider)    → Direct API to GSTN via GSP partner
  EaseMyGST / ClearTax          → Third-party filing platform
  NIC Portal API                → E-invoice + E-way bill
  Manual Export                 → JSON/Excel for offline GST portal filing

─── MODULE 14: E-INVOICE & E-WAY BILL ──────────────────────────
E-INVOICE:
  IRN Generation   : Auto via IRP (Invoice Registration Portal)
  QR Code          : Auto-embedded on invoice print (legally required)
  Cancellation     : Within 24 hours via NIC API + reason logging
  Threshold        : Above government-notified turnover limit
  Scope            : B2B transactions only (B2C exempt)
  Retry            : Auto retry + full error log for failed IRN

E-WAY BILL:
  Auto Generation  : GRN, inter-store transfer, sales > Rs.50,000
  Part B Update    : Vehicle number update after generation
  Cancellation     : Within 24 hours via API
  Extension        : For transit delays beyond validity period
  Bulk Generation  : Multiple invoices in one action
  Integration      : NIC E-Way Bill API (direct portal)

─── MODULE 15: TALLY SYNC ──────────────────────────────────────
TRANSACTION SYNC FLOW:
  Smriti OS → Tally Voucher Type
  Sales Invoice        → Sales Voucher
  Purchase/GRN         → Purchase Voucher
  Payment Received     → Receipt Voucher
  Payment Made         → Payment Voucher
  Credit Note          → Credit Note
  Debit Note           → Debit Note
  Stock Journal        → Stock Journal
  Day End Cash         → Cash Ledger Entry
  GST Summary          → GST Ledgers

SYNC METHODS:
  Tally XML Push   : Generate Tally XML → auto import via Tally gateway
  TallyPrime API   : Direct REST API (TallyPrime 3.0+ only)
  CSV/Excel Export : Manual import file (older Tally ERP 9 users)
  ODBC             : Direct DB connection (advanced/enterprise)

MASTER MAPPING:
  Customer Master  → Ledger (Sundry Debtors group)
  Supplier Master  → Ledger (Sundry Creditors group)
  Item Master      → Stock Item
  Category         → Stock Group
  Warehouse        → Godown
  Payment Mode     → Ledger (Bank or Cash)
  Tax Master       → Tax Ledger (GST groups)

SYNC FEATURES:
  Inventory Sync         : Real-time stock levels → all platforms (prevents oversell)
  Order Pull             : All platform orders → Smriti OS single fulfillment screen
  Price Sync             : Price changes → all platforms automatically
  Return Sync            : Platform returns → sales returns in inventory
  Product Listing        : Create listings from Smriti OS item master
  Dispatch Updates       : Shipment/tracking → pushed back to platforms
  Multi-warehouse        : Route orders to nearest/specified warehouse
  Settlement Reconcile   : Match platform settlements to Smriti OS sales

ARCHITECTURE:
  Platform Connectors   : REST API clients per platform
  Order Queue           : Celery + Redis (async order processing)
  Inventory Publisher   : WebSocket + Celery Beat (near real-time stock push)
  Error Handling        : Retry queue + Sentry alerts
  Reconciliation Engine : Custom Python module
