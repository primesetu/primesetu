Post-update Extensions – Procedure

Procedure to create post-update extensions

Creating Custom DLL and deploying it at POS location

To know how to create and deploy Custom DLL, Refer Appendix

Mandatory function to be present in a Custom DLL for Post-update Extensions is:

Public Function PostUpdateHook(RSData As Object)

The arguments to the function PostUpdateHook is an Object (RsData Object) with the following components.

Records to be inserted in General Lookup (GenLookUp) table

The Rec Id (Record Id) 13000 is maintained in GenLookUp table for configuring the customised DLL in post-update extensions.

In GenLookUp table, Code is inserted as a combination of Transaction Type, Operation Code and EXE Name each separated by a ‘+’ symbol. For example, if the Transaction Type is 1100, Operation Code is 01 and EXE Name is SR127000, then the corresponding Code is 1100+01+SR127000.

Alternatively, you can use Extension Script Generator Tool to generate the corresponding script which contains the SQL statements to be inserted in GenLookup table for configuring the required custom DLLs.
