Tally Interface Mapping

Shoper 9 can be configured to post the accounting and closing stock details to Tally 9 for generating the financial and inventory reports required by your organisation. To achieve this, you have to configure the details to be posted to Tally 9. This option allows you to do the necessary configuration to interface with Tally 9.

You can map many Showrooms to the same Tally Company. Each showroom can be mapped as a Cost Centre in a Tally company.

Note: The transaction created only in Shoper 9 can be posted to Tally.ERP 9.

 

Note:

1. Install Tally 9 and create a company, before setting up Tally Interface Mapping

2. Edit tally.ini file to add the lines:

    a. clientserver=<type>, where type can be client, server or both

    b. serverport=<port number>, use a port number between 9000 and 15000

    c. User TDL = Yes

    d. tdl = <path>\shoper9_T92.tcp if you are using Tally9 rel2.x; or tdl = <path>\shoper9_T93.tcp if you are using Tally9 rel 3.x, or tdl = <path>\shoper9_ERP.tcp if you are using Tally.ERP 9 where path is the location of the tcp file

 

How to reach: Setup > General > Tally Interface - Mapping

The Tally Interface - Mapping Manager window is displayed.

All existing mapping information for the selected showroom is displayed in the grid.

In this window you can execute the following functions:

Add: Click to create a new mapping for the selected showroom. Click here for more information.

Edit: Click to edit the mapping you have selected in the grid. Make necessary changes and save.

Exit: To abort the mapping process
