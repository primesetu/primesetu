Importing Pricing Master from Secondary Database to the Primary Database

The secondary database is repository of all the item master details applicable in a chain of stores. The primary database essentially consists of only those masters which are relevant for your installation. It is advisable to keep the primary database small and compact, so that your operations are fast and optimized. Use this option to selectively import item details from the secondary database to the primary database.

How to reach: Housekeeping > Data Import > Import Items from Secondary to Primary Db.

The Primary DataBase Updation window is displayed.

 

Use the search facility provided in this browse to filter the items as per the given parameters and load into the primary database. You can select multiple items and import it into the primary data base at one go.

Alt + R / F3: To Change Criteria

Alt + S / F4: To Start Search

Alt + W: To Toggle Focus

Alt + A: To open Advance Options window

Ctrl + Insert: To open previously searched details

 

Show All Records: Select to display all records in the Secondary Database.

Show n Records: Enter a number to display so many records at a time.

Sorting On: Select the field according to which the search results have to be sorted.

Order in Ascending: Select to sort in ascending order and un-select to sort on descending order.

Search: To display the search result

Advanced: For advanced search

Select All: To select all the search result

Clear: To clear all the selected result

Load: To update the pricing master details from the secondary database to the primary database

Exit: To quit the option.
