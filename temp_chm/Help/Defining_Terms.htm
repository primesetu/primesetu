Every Shoper 9 customer gets an account. This account is called a Tally.NET Account ID (frequently referred to as Account ID). This is basically a short and easily remembered name for your organisation, e.g., AcmeCorp. Every Account has one Account Administrator, recognized by an ID. An account with only one site (one serial number) is called a single site account. An account with multiple sites (more than one serial number) is called a multi site account. If you have multiple sites, then each site will have a Site ID. This is basically how you refer to the site, e.g., ABC Mall Bangalore or Retail Plaza Chennai. Every site will have one Site Administrator, recognized by an ID.

For a single site account, your Account ID and Account Administrator's ID will be created based on the e-mail ID provided during the licensing process. For example, ram@mail.com. This will be treated as the Account ID as well as the Account Administrator’s ID.

For a multi site account, your Account ID and Account Administrator's ID will be created based on your request. For example, AcmeCorp as the Account ID and admin@AcmeCorp.com as the Account Administrator’s ID. While activating any site, you need to provide the Account ID to identify the account to which the site belongs. During license activation for any site, you need to provide the Account ID, Site ID and an E-mail ID. This e-mail ID becomes the Site Administrator’s ID. For example, AcmeCorp as the Account ID, Crazy Fashions Bangalore as the Site ID and CrazyBlr@AcmeCorp.com as the Site Administrator’s ID.

The Account Administrator can create Tally.NET Users who can do administration activities, except license management. If such a Tally.NET User is linked to a site, then he can do administration activities for the site. In case the Tally.NET User is not linked to a site, he can do administration activities for all the sites in the account. Such Tally.NET Users get privileges based on the Security Level assigned to them.

Note: Any Tally.NET User can use Support Centre, for posting queries. He can also view the solutions provided to his queries. A Site Administrator can view queries posted by all Tally.NET Users in his site and provide solutions. In the case of a multi site account, the Account Administrator can view the queries posted by all the Tally.NET Users in all the sites linked to his account and provide solutions.
