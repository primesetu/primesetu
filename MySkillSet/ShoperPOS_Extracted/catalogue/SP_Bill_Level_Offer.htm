Bill Level Offer

This promotion type helps you to define schemes, which support bill level offers. Using this promotion type, you can define promotion schemes where selected items or a combination of different items are given free based on either bill value or billed quantity.

Based on:

Bill Value Range: Select if the offer varies according to the range of bill value. Click here to view the details captured, if the offer is based on Bill Value Range.

The Bill value range is captured as shown.

Enter the criteria to be satisfied by the total bill value to allow offer.

Bill Quantity Range: Select if the offer varies according to the range of billed quantity. Click here to view the offer details captured based on Bill Quantity Range.

The Bill quantity range is captured as shown.

Enter the criteria to be satisfied by the total bill quantity to allow offer.

Free Items

This section allows you to select the items to be given free. There are two options here.

  • Definition not based on item combination

    The Free Items section is as shown.

    Configure: Click to configure the options for item selection. Click here for more information.

    The item selection grid displays the item classification columns as configured. On pressing F2 in any item classification column, the corresponding browse window is displayed with the list of catalogued values to select from.

    Note: In the item selection grid:

       •  You can press F2 anywhere in the grid, irrespective of the order in which the item classification columns are present, to select values of any attribute.

       •  You have the flexibility to select either one or multiple attributes in a row, to specify the required items.

       •  If Stock No. of an item is selected, the remaining item classification columns for that stock number will be filled automatically.

    Quantity: Specify the quantity of the items that are given free. The same value is copied to all other rows.

  • Definition based on item combination

    The Free Items section is as shown.

    Bundle Items: Select if the offer is based on item combinations. On selecting this option, some more columns are added to the Items grid, where you need to specify the required item combination by grouping the items into different sets.

    Item Rate/Worth: Select if the item rate/ worth of the free item to be considered. When you select this, the possible comparison operators that can be used to specify the selection criteria are filled in the list. The choices are Rate =, Rate >, Rate <=, Rate Range, Worth =, Worth >, Worth <= and Worth Range. Select the appropriate one.

    Configure: Click to configure the options for item selection. Click here for more information.

    The item selection grid displays the item classification columns as configured. On pressing F2 in any item classification column, the corresponding browse window is displayed with the list of catalogued values to select from.

    Note: In the item selection grid:

       •  You can press F2 anywhere in the grid, irrespective of the order in which the item classification columns are present, to select values of any attribute.

       •  You have the flexibility to select either one or multiple attributes in a row, to specify the required items.

       •  If Stock No. of an item is selected, the remaining item classification columns for that stock number will be filled automatically.

    This promo is defined on item combinations, so you need to specify the quantity of the items and also rate criteria, if selected.

    In the Free Items grid, select the first set of items.

    Quantity: Specify the quantity for the first set of items.

    Set No.: Auto assigned depending on the condition selected in the next column.

    Condition: Select And or Or to specify the criteria. Last row will always have End. When you select And, the offer is applicable if the buyer picks the items in both the current row and the next row. Selecting Or indicates the offer is applicable if the buyer picks the item either in the current row or the next row.

    If the condition selected is And the item in the current row and next row will be in two different sets and the sets will be numbered accordingly. If the selection is Or, the items will be in the same set and both rows will be assigned the same set number.

    In case you are selecting Or, the quantity for both item combinations in the set will remain the same and hence the quantity column in the next row will be non editable.

    Notice that set numbers are assigned to the rows based on your selection in Condition.

Applied on: You may configure the default behaviour for selecting lowest/highest priced item to be given as free item.

Select Lowest priced item, if the free item is to be the lowest priced item among the items bought.

Select Highest priced item, if the free item is to be the highest priced item among the items bought.
