Buy Items and Free Items are Same

This promotion type helps you to define schemes where buy items and free items are same.

The details to be captured can be divided into two parts, namely, Buy Items and Offer Details.

Buy Items

This section allows you to select the items to be bought to avail offer.

The Buy Items section is as shown.

Configure: Click to configure the options for item selection. Click here for more information.

The Buy Items grid displays the item classification columns as configured. On pressing F2 in any item classification column, the corresponding browse window is displayed with the list of catalogued values to select from.

Note: In the Buy Items grid:

   •  You can press F2 anywhere in the grid, irrespective of the order in which the item classification columns are present, to select values of any attribute. Click here to learn more about value selection.

   •  You have the flexibility to select either one or multiple attributes in a row, to specify the required items.

   •  If Stock No. of an item is selected, the remaining item classification columns for that stock number will be filled automatically.

Offer Details

This section allows you to capture offer details for the items selected in Buy Items grid.

The Offer Details section is as shown.

Based on: Additional details can be captured based on item rate.

Item Rate: Select if the offer is based on item rate. When you select this, the possible comparison operators that can be used to specify the selection criteria are filled in the list. The choices are Rate =, Rate >, Rate <= and Rate Range. Select the appropriate one. Click here to view the offer details captured based on item rate.

Buy Group Details: Enter the quantity of the selected items to be bought in order to avail the offer. If offer is based on item rate, enter the criteria to be satisfied by rate and quantity of the selected items in order to avail the offer.

Free Group Details: Enter the item quantity that will be given free, if the buy group item quantity is purchased.

Offer Item: You may configure  the default behaviour for selecting lowest/highest priced item to be given as free item.

Lowest Priced: Select if the free item is to be the lowest priced item among the items bought.

Highest Priced: Select if the free item is to be the highest priced item among the items bought.

Alternate priced: Select this option to create combination of items during billing based on scheme definition and sort billed and free items in each combination.
